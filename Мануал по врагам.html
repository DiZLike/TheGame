🛠️ Система врагов: Архитектура и Создание

Это руководство описывает иерархию классов врагов и способы создания новых типов или дополнительных узлов на основе предоставленного кода.

🏗️ Иерархия классов

CharacterBody2D
├── BaseEnemy (base_enemy.gd)
│ ├── Здоровье, очки, взрыв
│ ├── ── StaticEnemy (static_enemy.gd)
│ │ ├── Интервальная атака
│ │ ├── ── Blazeman (бросает фаерболы)
│ │ ├── ── Flamethrower (огнемётчик)
│ │ ├── ── Turret_1 (турель с поворотом)
│ │ ├── ── SpitterEnemy (spitter_enemy.gd)
│ │ │ └── ── AcidSpitter (кислотный плеватель)
│ │ ├── ── AcidDripper (капает кислотой)
│ │ ├── ── DirectionalStaticEnemy (directional_static_enemy.gd)
│ │ │ ├── ── Camper (стреляет прямо, неуязвим в idle)
│ │ │ └── ── Trident (стрельба в 3 направлениях)
│ ├── ── MovingEnemy (moving_enemy.gd)
│ │ └── ── Schoolboy (ходит по платформам)
│ └── ── AnnoyingFlyer (annoying_flyer.gd, летает)

Area2D

    └── ── AcidDrip (acid_drip.gd, вертикальная капля)

📋 Как создать нового врага

Шаг 1: Выбор родительского класса

КлассПоведениеПример использования

StaticEnemyСтоит на месте, атакует по таймеруТурели, метатели
SpitterEnemyПодбрасывает снаряды вверх (дугой)Кислотные плеватели
DirectionalStaticEnemyСтреляет в фиксированных направленияхТрехнаправленная пушка

💡 Совет: Создавайте нового врага, наследуясь от наиболее подходящего класса. Не рекомендуется напрямую наследоваться от BaseEnemy, если только вам не нужно полностью уникальное поведение.

Шаг 2: Создание сцены (Scene)

  1. Создайте новую сцену с корневым узлом CharacterBody2D.
  2. Прикрепите скрипт вашего нового класса (например, my_new_enemy.gd).
  3. Добавьте необходимые дочерние узлы:
    • Спрайт: AnimatedSprite2D с именем MainSprite2D.
    • Коллизия: CollisionShape2D (обязательно).
    • Детектор экрана: VisibleOnScreenNotifier2D (обязательно, нужен для активации).
    • Для статичных врагов: Маркеры точек спавна снарядов (Marker2D), например ShootingPoint.
    • Для движущихся врагов: RayCast2D для проверки земли (GroundCheck) и Area2D для обнаружения стен (WallCheckArea).

Шаг 3: Написание скрипта

Ваш новый скрипт должен переопределить ключевые методы. Вот минимальный шаблон для нового статичного врага (похожего на турель):

extends StaticEnemy
class_name MyNewTurret

# 1. Настройки (свои уникальные переменные)
@export var bullet_scene: PackedScene = preload("res://path/to/my_bullet.tscn")
@export var custom_power: float = 1.5

# 2. Настройка характеристик (очки, тип атаки)
func _configure_stats() -> void:
    # Обязательно вызвать, если переопределяете
    super._configure_stats() 
    
    # Указываем паттерн для расчета очков
    _attack_pattern = "single" 
    _movement_type = "rotate"
    
    # Если нужен особый расчет здоровья
    health = 150

# 3. Настройка компонентов (точки спавна, таймеры)
func _setup_components() -> void:
    super._setup_components()
    # Найти свои уникальные узлы
    # my_node = $MySpecialNode

# 4. Переопределение атаки (ГЛАВНЫЙ МЕТОД)
func _execute_attack() -> void:
    if not bullet_scene:
        return
        
    var bullet = bullet_scene.instantiate()
    get_tree().root.add_child(bullet)
    bullet.global_position = $ShootingPoint.global_position
    
    # Установить направление, скорость и т.д.
    if bullet.has_method("set_shooter"):
        bullet.set_shooter(self)

🎯 Создание дополнительных классов снарядов

Если вам нужен новый тип пули, наследуйтесь от BaseEnemyProjectile.

Обязательные методы для переопределения:

  • _configure() — установка параметров по умолчанию (speed, damage, bullet_type).
  • _move(delta) — если снаряд движется не по прямой (как фаербол).
  • _update_visuals(delta) — поворот спрайта или анимации.
  • _play_hit_effect(target) — эффект при попадании.
extends BaseEnemyProjectile
class_name MyLaserBolt

func _configure() -> void:
    speed = 500.0
    damage = 2
    bullet_type = "laser" # важно для обработки урона
    life_time = 2.0
    # can_hit_player = true (уже по умолчанию)

func _update_visuals(delta: float) -> void:
    # Лазер всегда смотрит в направлении полета
    rotation = direction.angle()
    
    if animated_sprite:
        animated_sprite.play("move")
⚠️ Важно: Тип снаряда (bullet_type) используется в методе BaseEnemy._calculate_explosion_force() для определения силы взрыва врага. Убедитесь, что вы добавили обработку нового типа в этот метод, если хотите особый эффект при смерти врага.

⚙️ Ключевые методы для переопределения

МетодГде находитсяНазначение
_configure_stats()BaseEnemyУстановка _attack_pattern, _movement_type и health
_initialize()BaseEnemyПоиск игрока, начальные действия
_setup_components()BaseEnemyСоздание таймеров, привязка узлов
_execute_attack()StaticEnemy / DirectionalStaticEnemyСоздание снаряда/пули
_physics_process(delta)Разные классыДвижение, гравитация, поворот к игроку

💰 Система начисления очков

Очки рассчитываются автоматически на основе параметров, если переменная score равна 0.

SCORE = SCORE_BASE (50) 
      + (health * 1.0) 
      + ATTACK_BONUS (зависит от _attack_pattern) 
      + MOVEMENT_BONUS (зависит от _movement_type)

Вы можете задать score вручную в инспекторе сцены, тогда авто-расчет не сработает.

🏁 Полный пример: Враг "Стрелок"

Создадим врага, который стоит на месте и стреляет одиночными самонаводящимися пулями.

Скрипт (homer_shooter.gd):

extends StaticEnemy
class_name HomerShooter

@export var homing_bullet_scene: PackedScene

func _configure_stats() -> void:
    _attack_pattern = "homing"  # Даст бонус к очкам
    _movement_type = "none"
    health = 80

func _execute_attack() -> void:
    var bullet = homing_bullet_scene.instantiate()
    get_tree().root.add_child(bullet)
    bullet.global_position = global_position
    
    # Установить цель для самонаведения (если такой метод есть у пули)
    if bullet.has_method("set_target"):
        bullet.set_target(_player)
    
    if bullet.has_method("set_shooter"):
        bullet.set_shooter(self)

Структура сцены:

  • HomerShooter (CharacterBody2D)
    • MainSprite2D (AnimatedSprite2D)
    • CollisionShape2D
    • VisibleOnScreenNotifier2D
💡 Ресурсы по умолчанию: Обратите внимание, что hit_sound, death_sound и shot_sound уже предзагружены в BaseEnemy. Вы можете переопределить их в _ready() дочернего класса, если нужны уникальные звуки.

Следуя этому руководству, вы сможете легко расширять арсенал врагов, сохраняя консистентность кодовой базы и правильно интегрируя их в систему очков и эффектов.
